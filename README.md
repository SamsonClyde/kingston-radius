# Kingston Radius

Live music calendar within 90 minutes of Kingston, NY. Auto-updated every 2 hours from 50+ venues.

## How it works

A GitHub Actions workflow runs `scraper.py` every 2 hours. It scrapes 50+ venue websites and library APIs, writes the results to `events.js`, commits it, and Netlify auto-deploys the update. A separate Google Apps Script checks your Gmail for venue newsletters each morning at 6am and uses Claude to extract events from them.

```
scraper.py  →  events.js  →  git commit  →  Netlify deploy
Gmail newsletters  →  Apps Script + Claude  →  events.js  →  git commit  →  Netlify deploy
```

## Files

- `index.html` — the entire front end (HTML, CSS, JS in one file)
- `scraper.py` — scrapes all venues and writes `events.js`
- `events.js` — generated by the scraper; don't edit by hand
- `.github/workflows/scrape.yml` — runs the scraper every 2 hours
- `requirements.txt` — pinned Python dependencies
- `netlify/functions/save-events.js` — Netlify Function that lets the admin panel save edits directly to GitHub
- `KingstonRadiusEmailScraper.gs` — Google Apps Script for newsletter extraction

## Netlify environment variables

Set these in Netlify dashboard → Site settings → Environment variables:

| Variable | Description |
|---|---|
| `GITHUB_TOKEN` | Personal access token with `repo` scope |
| `GITHUB_OWNER` | Your GitHub username (e.g. `SamsonClyde`) |
| `GITHUB_REPO` | Repo name (e.g. `kingston-radius`) |
| `ADMIN_SECRET` | A strong random string — must match `ADMIN_SECRET` in `index.html` |

## Admin panel password

The admin password is stored as a SHA-256 hash in `index.html`. The current password is `kingston2025`.

To change it, run this in your browser console and replace the hash in `index.html`:
```js
crypto.subtle.digest('SHA-256', new TextEncoder().encode('yournewpassword'))
  .then(b => console.log([...new Uint8Array(b)].map(x=>x.toString(16).padStart(2,'0')).join('')))
```

## Adding events manually

Use the **⚙ button** on the live site. Events are saved in your browser. Click **Save to site** to push them directly to GitHub (requires `ADMIN_SECRET` to be configured).

## Adding a new venue to the scraper

**Option A — WordPress / Tribe Events site (most common)**

Add one line to `main()` in `scraper.py`:
```python
all_events += scrape_generic_tribe(
    "https://venuename.com/events/",
    "Venue Name", "City, NY",
    "https://maps.google.com/?q=Venue+Name+City+NY"
)
```

Then add the drive time from Tubby's to `DRIVE_TIMES` in `index.html`.

**Option B — Custom site structure**

Write a dedicated `scrape_venuename()` function following the pattern of existing scrapers, add it to `main()`, and add the drive time.

## Adding a venue newsletter to the email scraper

1. Open `KingstonRadiusEmailScraper.gs` in script.google.com
2. Add the sender's email or domain to `VENUE_SENDERS` in `CONFIG`
3. Run `setupTrigger()` once to re-save

## Running the scraper locally

```bash
pip install -r requirements.txt
python scraper.py
```

## Email scraper setup (one-time)

1. Go to [script.google.com](https://script.google.com), create a new project, paste `KingstonRadiusEmailScraper.gs`
2. Run `storeSecrets()` — it will prompt you for your Anthropic API key and GitHub token (stored securely, never in source)
3. Run `setupTrigger()` — schedules the daily 6am run
4. Done. Use `testSingleEmail()` to test against a real newsletter in your inbox
